## Overview

Konstantinos K supports seamless integrations with popular tools to streamline your documentation workflow. You can sync changes with version control systems, receive real-time notifications via webhooks, embed docs in other applications, and export content to various formats. These features help automate processes and extend functionality without custom development.

<Callout kind="info">
  Start with version control integration to keep your docs in sync with your codebase.
</Callout>

## Version Control Systems

Connect Konstantinos K to GitHub or GitLab to automatically sync documentation updates with your repository. This ensures version history and collaborative editing.

<Tabs>
  <Tab title="GitHub" icon="github">
    Create a personal access token with repo permissions.
    
    <Steps>
      <Step title="Generate Token" icon="key">
        Visit GitHub settings and create a token.
      </Step>
      <Step title="Connect in Dashboard" icon="settings">
        Navigate to Integrations in Konstantinos K dashboard at `https://dashboard.example.com/integrations`.
        
````bash
curl -X POST https://api.example.com/v1/integrations/github \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"repo": "your-org/your-repo", "token": "ghp_YOUR_TOKEN"}'
````
      </Step>
      <Step title="Sync Changes" icon="git-branch">
        Push updates to your repo branch to trigger auto-sync.
      </Step>
    </Steps>
  </Tab>
  <Tab title="GitLab" icon="git-branch">
    Use a project access token.
    
    <CodeGroup tabs="cURL,JavaScript">
````bash
curl -X POST https://api.example.com/v1/integrations/gitlab \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"project": "your-group/your-project", "token": "glpat-YOUR_TOKEN"}'
````
      ```javascript
      const response = await fetch('https://api.example.com/v1/integrations/gitlab', {
        method: 'POST',
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          project: 'your-group/your-project',
          token: 'glpat-YOUR_TOKEN'
        })
      });
      ```
    </CodeGroup>

</Tabs>

## Webhooks for Notifications

Set up webhooks to receive events like page updates or new versions. Point your webhook URL to `https://api.example.com/v1/webhooks`.

<ParamField path="webhook_url" param-type="string" required="true">
  Your endpoint to receive payloads.
</ParamField>

<ParamField header="X-Signature" param-type="string" required="false">
  HMAC signature for verification.
</ParamField>

<Request show-lines="true">
```bash
curl -X POST https://api.example.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "url": "https://your-webhook-url.com/webhook",
    "events": ["page.updated", "version.created"]
  }'
```
</Request>

<Response tabs="200,400">
```json
{
  "id": "wh_123",
  "status": "active",
  "events": ["page.updated"]
}
```

```json
{
  "error": "Invalid URL format"
}
```
</Response>

<Callout kind="tip">
  Verify webhook signatures using the shared secret to prevent unauthorized access.
</Callout>

## Embedding Documentation

Embed Konstantinos K pages in tools like Notion, Slack, or Confluence using iframes.

```html
<iframe
  src="https://docs.example.com/your-space/page-slug"
  width="100%"
  height="600"
  frameborder="0">
</iframe>
```

<Expandable title="Advanced Embed Options" default-open="false">
  Customize with query params: `?theme=dark&read-only=true`.
</Expandable>

## Exporting Content

Export docs to Markdown, PDF, or HTML for offline use or migration.

| Format    | Use Case                  | Endpoint Example                  |
|-----------|---------------------------|-----------------------------------|
| Markdown | Git repos, static sites  | `/export/markdown/{space-id}`    |
| PDF      | Printing, sharing        | `/export/pdf/{page-slug}`        |
| HTML     | Web embeds               | `/export/html/{space-id}`        |

```javascript highlight="2-4" show-lines={true}
const fs = require('fs');

async function exportMarkdown(spaceId) {
  const response = await fetch(`https://api.example.com/v1/export/markdown/${spaceId}`, {
    headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
  });
  const markdown = await response.text();
  fs.writeFileSync('docs.md', markdown);
}
```
