## Migration Guide

For v1.0.0 breaking changes, follow these steps to update your authentication setup.

<Steps>
  <Step title="Update Tokens" icon="key">

Generate a new Bearer token from your dashboard at `https://dashboard.example.com/tokens`.

  </Step>
  <Step title="Replace Headers" icon="code">

Update your API calls:

````javascript
// Old
fetch('https://api.example.com/docs', {
  headers: { 'Authorization': 'Basic ' + btoa('user:pass') }
});

// New
fetch('https://api.example.com/docs', {
  headers: { 'Authorization': `Bearer ${YOUR_TOKEN}` }
});
````

  </Step>
  <Step title="Test Integration" icon="check-circle">

Verify endpoints respond with `{200 OK}` status.

  </Step>
</Steps>

<Expandable title="Advanced Migration for Custom Integrations" default-open="false">

If you use webhooks, update the payload schema:

```json
{
  "event": "doc.updated",
  "data": {
    "id": "doc-123",
    "title": "Updated Doc"
  }
}
```

Ensure your endpoint at `https://your-webhook-url.com/webhook` handles the new `{schema_version: "2.0"}` field.

</Expandable>
